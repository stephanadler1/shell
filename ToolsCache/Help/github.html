github(1), so(1), vso(1) Manual Page

Name

Various commands to open software development oriented web sites and their subpages.

  • github – Opens Github or a subpage in it.
  • so – Opens Stackoverflow or a subpage in it.
  • vso – Opens Azure DevOps (fka. Visual Studio Online) or a subpage in it.

Synopsis

github [reponame]

github [/username[/reponame]]

so [tagname]

vso [reponame]

Description

Let's assume the environment variable TOOLS_URL_GITHUB is defined as https://github.com/grpc/. When using github it opens this page. When calling github grpc it opens the grpc repository inside that user. When calling github /microsoft it opens the microsoft user page. github /microsoft/vscode takes you to the vscode repository of the microsoft user.

When calling so it opens the configured default page, which at the moment are questions tagged 'C#'. By using so go you can open Stackoverflow for questions tagged 'go'.

Let's assume the environment variable TOOLS_URL_VSO is defined as https://microsoft.visualstudio.com/. When using vso it opens this page. When calling vso grpc it opens the grpc repository inside that account.

  • Github
  • Visual Studio Online

Environment

TOOLS_URL_GITHUB
If defined enables the github command. This environment variable should contain the most used Github url, e.g. https://github.com/youraccountname/.
TOOLS_URL_VSO
If defined enables the vso command. This environment variable should contain the most used Visual Studio Online url, e.g. https://youraccountname.visualstudio.com/.
